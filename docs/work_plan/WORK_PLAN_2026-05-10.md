## Priority order

### 🔥 HIGH — Bundle bloat fixes (~5 MB savings, no code changes)

#### 1. ✅ Exclude `*.symbols` files from Firebase Hosting deploy
- **Why**: `canvaskit/*.symbols` are ~4 MB of debug symbols that should
  never be served to users. Currently Firebase Hosting serves them
  because they're in the Flutter SDK output.
- **Action**: edit `firebase.json` `hosting.ignore` array
- **Estimated savings**: ~4 MB raw

#### 2. ✅ Audit `pubspec.yaml` for stale Stripe-related deps
- **Why**: CLAUDE.md §2 / §4 says Stripe was removed in v11.9.x but
  some package deps may still be in `pubspec.yaml` adding to bundle.
- **Action**: `grep stripe pubspec.yaml` + check if any deps are dead
- **Estimated savings**: ~200 KB

#### 3. ✅ Verify Brotli compression on Firebase Hosting
- **Why**: Firebase Hosting auto-applies Brotli to text assets if the
  client supports it. Saves 25%+ on top of gzip. Need to verify it's
  actually active.
- **Action**: `curl -H "Accept-Encoding: br" -I` against production URL
- **Output**: confirmation note in the bundle analysis MD

#### 4. ✅ Convert `NEW_LOGO1.png.png` → optimized format
- **Why**: 604 KB single PNG with double extension (typo).
- **Action**: rename properly. WebP conversion deferred (requires `cwebp`
  binary) — but rename + resize via Flutter's tool is doable.
- **Estimated savings**: 200-500 KB depending on tool availability
