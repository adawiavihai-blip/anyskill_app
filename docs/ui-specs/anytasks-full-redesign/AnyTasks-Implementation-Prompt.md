# פרומפט ליישום שדרוג מודול AnyTasks – אפליקציית AnySkill

## הקדמה כללית

אנחנו משדרגים את מודול המשימות (AnyTasks) באפליקציית AnySkill.
המודול הוא חלק מאפליקציית marketplace דו-צדדית (לקוחות ← נותני שירות).
המסכים הנוכחיים לא מקצועיים ולא יעילים. אנחנו רוצים לשדרג ברמה של TaskRabbit, Thumbtack, Airtasker.

להלן כל השינויים הנדרשים — כולל הסרות, שינויי UI/UX, ו-4 פיצ'רים חדשים.

---

## חלק א': הסרות ושינויים במסכים הקיימים

### מסך ראשי (Home / רשימת משימות)

1. **להסיר** את שורת החיפוש "תאר מה אתה צריך ו-AI ימצא את האיש הנכון..." — לא רלוונטי למסך הזה.
2. **להסיר** את אייקון המזוודה בצד שמאל של ה-Header — זה שייך לנותן שירות, לא ללקוח.
3. **להוסיף חץ חזרה** (back navigation) בכל מסך — כרגע אין אפשרות לחזור אחורה.
4. **לשדרג את ה-Header** — עיצוב מודרני עם gradient כהה (dark navy/slate), שם המשתמש + אווטאר, וברכה "מה צריך לעשות היום?".
5. **קטגוריות מהירות** — שורת אייקונים אופקית מתחת ל-Header עם הקטגוריות הראשיות (משלוחים, ניקיון, תיקונים, הובלות, טיפול בחיות).
6. **כרטיס משימה פעילה** — לשדרג את כרטיס המשימה הקיים: להוסיף סטטוס צבעוני (פתוחה/בביצוע/הושלמה), מספר הצעות שהתקבלו, וכפתורי "צפה בהצעות" + "פרטי המשימה".

### מסך פרסום משימה חדשה

7. **להסיר** את הבאנר הכחול-סגול "לחץ למילוי אוטומטי עם AI" — מיותר וגורם לבלגן ויזואלי.
8. **להחליף** את "תקציב בש״ח" ➜ **"תשלום על המשימה"**.
9. **להסיר** את "תאריך יעד (רשות)" — כבר יש כפתורי דחיפות שמטפלים בזה.
10. **לפצל את הטופס ל-4 שלבים** (wizard/stepper) עם Progress Bar:
    - **שלב 1: קטגוריה** — גריד של 10 קטגוריות עם אייקונים (3 בשורה)
    - **שלב 2: פרטים** — כותרת קצרה (עד 100 תווים), תיאור מפורט (עד 2000 תווים), העלאת תמונה (רשות)
    - **שלב 3: תשלום ולוגיסטיקה** — תשלום על המשימה, דחיפות, מיקום (מאיפה/לאיפה), סוג הוכחה, checkbox "משימה מרחוק"
    - **שלב 4: סיכום** — הצגת כל הפרטים לפני פרסום + הודעת אבטחת תשלום (escrow)
11. **כפתורי תקציב מהירים** — כפתורי סכום מוכנים (₪50, ₪80, ₪100, ₪150, ₪200, ₪300, ₪500) בנוסף לשדה הזנה ידנית.
12. **3 רמות דחיפות** עם אייקונים: 🕐 גמיש | ⚡ היום | 🔥 דחוף עכשיו.
13. **3 סוגי הוכחה**: 📷 תמונה | 📝 טקסט | 📋 תמונה + טקסט.

## חלק ב': פיצ'ר #1 — הצעות מחיר חכמות (Smart Pricing)

### מה זה
כשהלקוח מגיע לשלב התשלום, המערכת מציגה לו טווח מחירים מומלץ בהתבסס על הקטגוריה שבחר.

### איך זה עובד
- כל קטגוריה מכילה `priceRange` — למשל: משלוחים [30–120], ניקיון [80–250], הובלות [200–800].
- מוצג באנר כחול בהיר עם אייקון 💡 שכולל:
  - טקסט: "משימות דומות ב[קטגוריה] עלו בין ₪X ל-₪Y"
  - פס התקדמות (progress bar) שמראה איפה הסכום שהלקוח הזין נמצא ביחס לטווח
  - הצבע משתנה: כתום אם נמוך מדי, ירוק אם בטווח, אדום אם גבוה מדי
  - טיפ טקסטואלי:
    - מחיר נמוך: "⚠️ מחיר נמוך — ייתכן שתקבל פחות הצעות"
    - מחיר בטווח: "✅ מחיר מצוין — צפי להצעות רבות"
    - מחיר גבוה: "🔝 מחיר גבוה — תקבל את נותני השירות הטובים ביותר"
- כפתורי הסכום המהירים מתעדכנים דינמית לפי הקטגוריה

### מקור הנתונים
- בשלב ראשון: טווחים סטטיים per-category (hardcoded)
- בעתיד: חישוב דינמי מהיסטוריית משימות אמיתיות

## חלק ג': פיצ'ר #3 — הצעות בזמן אמת (Real-Time Offers)

### מה זה
מסך שמוצג מיד אחרי פרסום משימה, עם אנימציית חיפוש ונותני שירות שמגיעים אחד-אחד.

### Flow
1. **שלב חיפוש**: ספינר מרכזי עם אייקון 🔍, טקסט "מחפשים נותני שירות...", ומונה שעולה ("סורקים 12 נותני שירות באזור שלך")
2. **הצעות מגיעות**: כרטיסי נותן שירות מופיעים אחד אחרי השני עם אנימציית slide-in
3. **באנר עליון**: נקודה ירוקה פועמת + "3 הצעות התקבלו — עוד מגיעות..."

### כרטיס נותן שירות כולל
- אווטאר + שם
- דירוג (כוכבים + מספר ביקורות)
- מספר משימות שהושלמו
- מחיר מוצע + זמן הגעה משוער
- Badge (אופציונלי): "⭐ מומלץ", "🚀 מהיר"
- כפתור "בחר נותן שירות ✓" (ירוק, בולט)
- כפתור צ'אט (💬)

### טכני
- לקשר ל-WebSocket / Realtime DB (Firebase/Supabase) לקבלת הצעות בזמן אמת
- Fallback: polling כל 5 שניות
- Push notification כשמגיעה הצעה חדשה

## חלק ד': פיצ'ר #7 — צ'אט עם Quick Replies

### מה זה
מסך צ'אט מלא בין הלקוח לנותן השירות, עם כפתורי תגובה מהירה.

### עיצוב
- **Header**: אווטאר + שם נותן השירות + סטטוס "מחובר עכשיו" (נקודה ירוקה) + חץ חזרה
- **בועות הודעות**: הלקוח בירוק (gradient), נותן השירות בלבן, עם חותמת זמן
- **Quick Replies**: 4 כפתורים אופקיים מעל שדה הקלט:
  - 🕐 "מתי אפשר להתחיל?"
  - 💪 "יש לך ניסיון בתחום?"
  - 💰 "אפשר לדבר על המחיר?"
  - 📋 "אפשר עוד פרטים?"
- **שדה הקלט**: placeholder "הקלד הודעה...", כפתור שליחה ירוק

### טכני
- Real-time messaging (Firebase Firestore / Supabase Realtime / Socket.io)
- Push notifications להודעות חדשות
- Quick Replies הם כפתורים שפשוט שולחים את הטקסט כהודעה רגילה
- אפשרות שליחת תמונות (בעתיד)

## חלק ה': פיצ'ר #8 — מעקב משימה חי (Live Task Tracker)

### מה זה
מסך Timeline שמציג את סטטוס המשימה בזמן אמת, בסגנון Uber.

### 6 שלבים בציר הזמן
1. 📢 **פורסמה** — המשימה פורסמה ומחכה להצעות
2. 🤝 **נותן שירות קיבל** — נותן שירות נבחר וקיבל את המשימה
3. 🚗 **בדרך אליך** — נותן השירות יצא לדרך
4. ⚡ **בביצוע** — המשימה מתבצעת כרגע
5. ✅ **בוצע** — נותן השירות סיים ושלח הוכחת השלמה
6. ⭐ **דורג** — שני הצדדים דירגו

### עיצוב
- Timeline אנכי עם קו מחבר
- שלב פעיל: נקודה גדולה צבעונית עם צל + נקודה פועמת + "עכשיו"
- שלבים שהושלמו: checkmark ✓ + קו ירוק
- שלבים עתידיים: אפורים ושקופים (opacity 0.4)
- **כרטיס נותן שירות** בחלק העליון: אווטאר, שם, דירוג, כפתור "💬 צ'אט"
- **כפתור דירוג**: מופיע רק כשהמשימה מסומנת כ"בוצע" — "⭐ דרג את נותן השירות"

### טכני
- סטטוס מתעדכן בזמן אמת מה-DB
- Push notification על כל שינוי סטטוס
- נותן השירות מעדכן סטטוס מצידו (כפתור "התחלתי" / "סיימתי" / "העלה הוכחה")

## חלק ו': פיצ'ר #10 — דירוג כפול (Airbnb Style Dual Rating)

### מה זה
מערכת דירוג הדדית — גם הלקוח מדרג את נותן השירות וגם ההפך.
הדירוגים נחשפים רק אחרי ששני הצדדים סיימו לדרג (בדיוק כמו Airbnb).

### הערה חשובה
מערכת הדירוג הזו כבר קיימת באפליקציה בסגנון Airbnb עבור חלקים אחרים. צריך להרחיב את אותה מערכת קיימת כך שתחול גם על מודול AnyTasks — לא לבנות מערכת חדשה.

### מסך הדירוג כולל

**חלק 1: פרטי נותן השירות**
- אווטאר גדול + שם + שם המשימה

**חלק 2: דירוג כוכבים**
- 5 כוכבים עם hover effect (הגדלה + צבע)
- תווית דינמית: גרוע / לא טוב / סביר / טוב / מעולה!
- צבע התווית משתנה: אדום (1-2), כתום (3), ירוק (4-5)

**חלק 3: תגיות מהירות (מופיעות אחרי בחירת כוכבים)**
- כפתורי chip שאפשר לסמן כמה:
  - דייקן | מקצועי | אדיב | נקי ומסודר | מהיר | תקשורת טובה

**חלק 4: ביקורת חופשית**
- textarea עם placeholder "ספר לאחרים איך עבר השירות..."

**חלק 5: הודעת דירוג הדדי**
- באנר צהוב: "⭐ דירוג הדדי בסגנון Airbnb — גם [שם] מדרג אותך. הדירוגים נחשפים רק אחרי ששניכם סיימתם — כך שניכם כנים יותר."

**מסך הצלחה (אחרי שליחה)**
- אנימציית 🎉
- "תודה על הדירוג!"
- הודעה: "גם [שם] ידרג אותך — הדירוג ההדדי יופיע לשניכם"
- כפתור "חזרה למשימות"

### לוגיקת Backend
- שמירת דירוג בטבלה: `task_id`, `rater_id`, `rated_id`, `stars`, `tags[]`, `review_text`, `created_at`
- שדה `revealed: boolean` — מתחיל כ-false
- Trigger: כששני הדירוגים קיימים ל-task_id, לעדכן revealed=true לשניהם
- API: `GET /tasks/{id}/rating` — מחזיר את הדירוג רק אם revealed=true

## חלק ז': הנחיות עיצוב כלליות

### צבעים
- Primary: #10B981 (ירוק — כפתורים ראשיים, סטטוס פעיל)
- Dark: #0F172A (header, טקסט ראשי)
- Background: #FAFBFC
- Card: #FFFFFF עם shadow: `0 2px 16px rgba(0,0,0,0.04)`
- Border: #E2E8F0
- Text secondary: #64748B
- Text muted: #94A3B8

### טיפוגרפיה
- Font: Rubik (Google Fonts) — תומך בעברית מצוין
- כותרות: 700-800 weight
- גוף: 400-500 weight
- גדלים: כותרת מסך 22px, כותרת כרטיס 15-16px, גוף 14px, מטא 12-13px

### אנימציות
- כניסת אלמנטים: `translateY(16px) → 0` + `opacity 0→1`, duration 0.3-0.5s, stagger 0.04-0.08s
- מעברי שלבים: `translateX(-20px) → 0` + fade
- כפתורים: `transition: all 0.2s`
- נקודה פועמת (live): `animation: pulse 1.5s infinite`

### כיוון
- כל המסכים ב-RTL (direction: rtl)
- אייקונים וטקסט מיושרים לימין
- חץ חזרה → (חץ ימינה ב-RTL)

### רדיוסים
- כרטיסים: 20px
- כפתורים: 12-16px
- Input fields: 14px
- Badges/chips: 20px
- אווטארים: 16px
