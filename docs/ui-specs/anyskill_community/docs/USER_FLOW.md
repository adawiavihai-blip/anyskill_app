# 🔄 User Flow - זרימת המשתמש

מסמך זה מתאר את כל הזרימות העיקריות של "קהילה" - מהיכן המשתמש נכנס, אילו פעולות הוא יכול לבצע, ואיך הן קשורות זו לזו.

---

## 👥 דמויות (Personas)

### דמות 1: רחל - הפונה (Requester)
- אישה בת 78
- צריכה עזרה בקניות / שיעורים / תיקונים
- לא טכנולוגית במיוחד
- רוצה: לקבל עזרה מהר ובצורה אנושית

### דמות 2: דניאל - המתנדב (Volunteer)
- בן 35, חשמלאי במקצועו
- רוצה: לעזור ולקבל הוקרה (לב זהב + קידום בחיפוש)
- כבר מתנדב באופן קבוע

### דמות 3: מיכל - המשתמש החדש
- בת 28, מורה למתמטיקה
- רוצה: להבין אם זה שווה את הזמן שלה
- שואלת: "מה אני מקבלת בתמורה?"

## 🆕 מסע משתמש חדש (First-Time Volunteer)

הפעם הראשונה שמשתמש משלים התנדבות:

```
[חגיגת השלמת התנדבות הראשונה]
                  ↓
      [לב זהב ראשון - מסך מיוחד]
   • "דניאל, קיבלת את הלב הראשון."
   • הסבר: "יוצג למשך 30 יום"
   • 3 תועלות:
     ✓ סימן אמון על הפרופיל
     ✓ קידום בחיפוש למשך 30 יום
     ✓ פתקי תודה אמיתיים
                  ↓
      הצג את הפרופיל החדש שלי

[פרופיל אישי]
   • לב זהב חדש על האווטאר!
   • באנר: "לב זהב פעיל · עוד 30 ימים"
   • Badge: "Starter" (קבוע - 1 התנדבות הושלמה)
```

## 🔁 חידוש לב הזהב (Streak Renewal)

```
יום 1:    משתמש משלים התנדבות
          → goldHeartExpiresAt = יום 31
          → לב זהב פעיל

יום 5:    בודק את הפרופיל
          → "לב זהב פעיל · עוד 26 ימים"

יום 10:   משלים התנדבות נוספת!
          → goldHeartExpiresAt = יום 40 (חידוש)
          → "לב זהב פעיל · עוד 30 ימים"
          ↑ הספירה התחילה מחדש

יום 25:   בודק שוב
          → "לב זהב פעיל · עוד 15 ימים"

יום 40:   ⏰ הלב פג!
          → goldHeartExpiresAt < now()
          → לא מציגים את הלב יותר
          → Search rank -50 חוזר לרגיל
```

## ❌ ביטולים ומקרים מיוחדים

### תרחיש: דניאל לא הגיע
```
דניאל תפס התנדבות → לא הגיע
                          ↓
       רחל לוחצת "המתנדב לא הגיע"
                          ↓
       ההתנדבות מבוטלת, אין הענקת לב זהב
                          ↓
   דניאל מקבל צמצום במשקל "מוניטין" (אפליקטיבי)
```

### תרחיש: זמן מינימום לא מולא
```
דניאל סיים אחרי 7 דקות (מינימום 15)
                  ↓
   המערכת חוסמת את כפתור "סיימתי"
   ומציגה: "מינימום 15 דקות"
                  ↓
   דניאל ימתין עוד 8 דקות
```

### תרחיש: ביקורת קצרה מ-10 תווים
```
רחל כותבת "תודה" (5 תווים)
              ↓
   כפתור "אשר" כבוי
   טקסט בכתום: "5 / 10 מינ'"
              ↓
   רק אחרי 10+ תווים - ניתן לאשר
```

### תרחיש: cooldown של 30 יום
```
דניאל עזר לרחל ב-1/05
            ↓
דניאל לא יכול לעזור לרחל שוב לפני 31/05
   (מניעת ניסיונות "בייעוץ פנימי" / שותפות)
            ↓
ב-31/05 - הוא יכול שוב
```

## 📊 דחיפויות (Urgency Levels)

```
🔴 דחוף    → תוך 47 דקות / שעה / היום
            (עדיפות עליונה במסך, בולט באדום)

🟡 השבוע   → תוך 7 ימים
            (פילטר רגיל, ללא הדגשה)

⚪ רגיל    → ללא דדליין
            (ברירת מחדל)
```

## 🎯 KPIs למדידה

### למתנדב
- זמן ממסך הבית עד תפיסת בקשה: **יעד < 60 שניות**
- אחוז מתנדבים שמסיימים את ההתנדבות הראשונה: **יעד > 70%**
- אחוז מתנדבים עם לב זהב פעיל ברגע נתון: **יעד > 25%**

### לפונה
- זמן ממיתן בקשה ועד תפיסה: **יעד < 30 דקות**
- אחוז בקשות שלא קיבלו מענה: **יעד < 15%**
- אחוז ביקורות שניתנו: **יעד > 90%**

### למערכת
- שביעות רצון ממוצעת (כוכבים): **יעד > 4.7**
- חידושי לב זהב חוזרים: **יעד > 40%**
- אחוז משתמשים שהפכו ל-"Pillar" (5+ התנדבויות): **יעד > 8%**

## 🎁 רגעים חשובים (Magic Moments)

מסכים שדורשים תשומת לב מיוחדת בעיצוב/UX:

1. **מסך 06 - חגיגת השלמת התנדבות** - כל פעם
2. **מסך 15 - לב זהב ראשון** - פעם בחיים
3. **מסך 11 שקופית 2 - הסבר על הלב** - פעם בחיים
4. **מסך 12 - סיכום שנתי** - פעם בשנה (1 בינואר)
5. **מסך 16 - התראת push** - הרגע שמושך אותו חזרה לאפליקציה
