# משימה: בניית פיצ'ר "גרר אופנועים" (Motorcycle Towing) באפליקציה

## הקשר כללי

אני בונה אפליקציה ל-marketplace של נותני שירות (similar to Honeybook/Thumbtack מותאם לישראל). הקטגוריה הראשית היא "שירותי רכב ותחבורה" ובתוכה התת-קטגוריה "גרר אופנועים".

המטרה: כשנותן שירות בוחר בתת-הקטגוריה "גרר אופנועים", צריך להופיע לו בלוק חכם ייעודי בעמוד עריכת הפרופיל. הבלוק מסתנכרן אוטומטית לפרופיל הציבורי שהלקוח רואה.

## שפה וכיוון

- **השפה: עברית בלבד**
- **כיוון: RTL (right-to-left)**
- **קהל יעד נותני שירות: גברים ברובם — כל הניסוחים בלשון זכר** ("סמן", "תוכל", "מטפל בהן", וכו')

## ארכיטקטורה — 4 ממשקים נפרדים

יש לבנות 4 ממשקים, כל אחד בקובץ נפרד או כמסך נפרד:

### 1. **Admin Dashboard** — ניהול קטגוריה (`admin-bike-types-management`)
פאנל ניהול שלי (כמנהל המערכת) שבו אני מעלה תמונות וסוגי אופנועים שיוצגו לכל נותני השירות. **רק המנהל** יכול להעלות/לערוך/למחוק תמונות.

### 2. **Provider Edit Profile** — עריכת פרופיל נותן שירות (`provider-tow-edit-block`)
הבלוק שמופיע לנותן השירות אחרי שבחר בתת-קטגוריה "גרר אופנועים". כאן הוא ממלא את כל פרטי השירות שלו פעם אחת.

### 3. **Customer Profile View** — תצוגת לקוח בפרופיל (`customer-tow-profile-block`)
איך הבלוק נראה ללקוח כשהוא צופה בפרופיל הציבורי של נותן השירות. **ממוקם בין סקציית "אודות" לסקציית "השירות"** בעמוד הפרופיל הקיים.

### 4. **Booking Flow** — זרימת הזמנה (`booking-flow-screens`)
5 שלבים + מסך אישור + מסך מעקב חי. נפתח כשהלקוח לוחץ על כפתור הזמנה (קיים כבר במערכת).

## מה כבר קיים באפליקציה (אל תיגע!)

הדברים האלו כבר ממומשים במערכת הקיימת — **אל תוסיף אותם בבלוק החדש**:
- ❌ מסמכי אימות / רישיונות (קיים בעריכת פרופיל הכללית)
- ❌ תמונת רכב הגרר (קיים בגלריית הנותן שירות)
- ❌ מעקב GPS חי (קיים כפיצ'ר מערכת)
- ❌ יומן זמינות (קיים מתחת לכל בלוק)
- ❌ כפתורי "הזמן" / "שאל שאלה" / "הצעת מחיר" (קיימים כפיצ'ר גלובלי)

### Typography
- Font: System default sans-serif (יתאים גם לעברית)
- Font weights: 400 (regular) ו-500 (medium). **אל תשתמש ב-bold (700)** — נראה כבד מדי.
- Letter-spacing on labels: 0.5px

### Spacing & Layout
- Border radius: 6px (small), 8px (medium), 12px (large)
- Padding בכרטיסים: 14-16px
- Gap בין סקציות: 12-16px
- מקסימום רוחב מובייל: 100% של viewport

### עקרונות עיצוב
1. **Minimal but informative** — כל פיקסל צריך להעביר ערך
2. **Soft shadows** — צללים עדינים, לא דרמטיים
3. **Subtle gradients** — רק ב-Hero header
4. **Animations** — רק במקומות שמעבירים מצב חי (פולס על "זמין כעת")
5. **No emojis** בכפתורי פעולה (נותן מראה לא מקצועי)
6. **שימוש ב-SVG** במקום אייקונים מספריות חיצוניות

## מפרט פונקציונלי מפורט

### **Admin Dashboard - סוגי אופנועים**

**כותרת:** "ניהול קטגוריה: גרר אופנועים"

**Stats Cards (3):**
- סוגי אופנועים פעילים (X +Y השבוע)
- נותני שירות בקטגוריה (X)
- סוגים ללא תמונה (X) — בצבע אזהרה אם יש

**Bike Type Management Grid (2 columns):**
כל סוג אופנוע:
- תמונה גדולה 16:9 (מקור: ספריית מערכת או Stock או URL)
- שם הסוג
- מטא: "X נותני שירות מסמנים את הסוג הזה"
- תג סטטוס (פעיל/לא פעיל)
- בריחוף: כפתור "החלף תמונה" + "מחק"
- כפתור "+ הוסף סוג חדש" בסוף הרשת

**רשימת סוגים התחלתית (יש להגיע איתה כברירת מחדל):**
1. ספורט (Sport)
2. קרוזר (Cruiser)
3. אדוונצ'ר (Adventure)
4. קטנוע (Scooter)
5. אופנועי שטח (Off-road)
6. וינטג' (Vintage)

**מודל העלאת תמונה:**
- העלה קובץ (JPG/PNG עד 5MB)
- ספריית מערכת
- חיפוש Stock
- URL חיצוני

### **Provider Edit Block - הבלוק שנותן השירות ממלא**

**כותרת ראשית:** "הגדרות גרר אופנועים"
**תיאור:** "פרטי השירות שיוצגו ללקוחות"
**Progress bar:** מציג אחוז השלמה (0-100%)

**6 סקציות:**

#### סקציה 1: סוגי אופנועים שאתה גורר (חובה)
- רשת 3 עמודות
- כל סוג: תמונה (קבועה ממנהל) + שם + checkbox קטן בתחתית
- נותן השירות **לא יכול לשנות תמונה**, רק לסמן/לבטל
- מצב נבחר: רקע סגול בהיר + border סגול + checkbox מסומן

#### סקציה 2: תמחור (חובה)
- שדה: "מחיר בסיס" (₪) — ברירת מחדל 180, hint: "מינימום קריאה (כולל 10 ק"מ ראשונים)"
- שדה: "מחיר לק"מ נוסף" (₪) — ברירת מחדל 4.5
- **קופסה מיוחדת לתוספת לילה:**
  - אייקון ירח
  - שדה אחוזים (₪)
  - שדה שעת התחלה (`<input type="time">`) — ברירת מחדל 22:00
  - שדה שעת סיום (`<input type="time">`) — ברירת מחדל 06:00
  - הסבר: "תופעל אוטומטית בשעות שתגדיר ובשבתות"
- שדה: "תוספת חירום מיידי" (%) — ברירת מחדל 50, hint: "לקריאות עם דרישת הגעה מתחת ל-30 דקות"

#### סקציה 3: ציוד ושיטת גרירה (אופציונלי)
טוגלים עם הסבר תחתי לכל אחד:
- משאית פלטה (Flatbed) — "השיטה הכי בטוחה — האופנוע מורם לחלוטין מהקרקע"
- עריסת גלגל קדמי (Wheel Cradle) — "לאופנועי ספורט עם פיירינג נמוך"
- רצועות בד רכות (Soft Straps) — "לא פוגעות בכרום, צבע ופיירינג"
- כננת חשמלית — "לאופנועים עם מנוע תקוע או גיר נעול"
- דולי עגלה (Tow Dolly) — "למרחקים קצרים — גלגל אחורי על הכביש"

#### סקציה 4: סוגי קריאות שאתה מטפל בהן (אופציונלי)
Pills מולטי-סלקט:
- תאונות דרכים, תקלות מנוע, פנצ'ר, מצבר מת, גרירה מתוכננת, חילוץ משטח/בוץ, דלק שגוי, נעילת מפתחות, העברה בין-עירונית

#### סקציה 5: אזור פעילות (חובה)
**2 טאבים:**

**טאב A: רדיוס מהבסיס**
- שדה כתובת בסיס
- סליידר רדיוס (5-200 ק"מ)
- מפה אינטראקטיבית עם עיגול שמתעדכן חי
- לג'נד: "אזור שירות · X קמ"ר" + "Y ערים בכיסוי"

**טאב B: ציור אזור על המפה**
- באנר הסבר צהוב
- מפה עם כלי "צייר" / "נקה"
- אפשרות לחיצה על נקודות לציור פוליגון
- כפתור "סיים ציור ✓"

#### סקציה 6: תכונות חכמות
טוגלים:
- תמונות "לפני/אחרי" אוטומטיות — "המערכת תזכיר לך לצלם בכל גרירה — מגן עליך מפני תלונות"
- הצעת מחיר מיידית — "מחושבת אוטומטית מהמחירים שמילאת למעלה"
- צ'אט פנימי עם הלקוח — "תקשורת מאובטחת — בלי לחשוף מספר טלפון"

**Footer:**
- כפתור "תצוגה מקדימה" — מציג איך הלקוח יראה
- Save bar דביק (sticky): "ביטול" + "שמור ופרסם בפרופיל"
- אינדיקטור "נשמר אוטומטית" בראש הדף

### **Customer Profile View - מה שהלקוח רואה**

**מיקום:** בין "אודות" ל"השירות" בפרופיל הקיים.

**Hero Header (גרדיאנט סגול):**
- כותרת: "שירות גרר אופנועים" עם אייקון
- תג "זמין כעת" עם נקודה ירוקה פועמת (אם הנותן שירות זמין)
- 3 כרטיסי מידע: זמן הגעה (22-35 דקות), מחיר התחלתי (₪180 + ₪4.5/ק"מ), רדיוס שירות (50 ק"מ)

**5 סקציות:**

1. **סוגי אופנועים** — סקרול אופקי של תמונות עם שמות (רק הסוגים שהנותן שירות סימן)
2. **שיטת גרירה והציוד** — רשת 2x2 עם וי ירוק (רק מה שמופעל)
3. **סוגי קריאות** — Pills סגולים (רק מה שסומן)
4. **תמחור שקוף** — קופסה צהבהבה חמה עם פירוט מלא ושעות לילה
5. **אזור פעילות** — מפה ויזואלית + סטטיסטיקה

**Trust Strip (סיום ירוק):**
4 תגי אמון: מאומת, ביטוח מלא לאופנוע, תמונות לפני/אחרי, צ'אט מאובטח

**אין כפתורי CTA** — הם קיימים גלובלית במערכת.

### **Booking Flow - זרימת הזמנה (5 שלבים)**

**שלב 1: סוג אופנוע**
- רשת תמונות לבחירה (single-select)
- שדה "דגם" אופציונלי

**שלב 2: סיבת הגרירה**
- 6 אפשרויות עם אייקונים: תאונה, תקלת מנוע, פנצ'ר, מצבר מת, גלגלים נעולים, העברה מתוכננת
- שדה "פרטים נוספים" אופציונלי

**שלב 3: מיקום ותמונות**
- מפה עם פין ירוק (מאיפה) ופין סגול (לאן)
- שדה כתובת מאיפה (אוטו-זיהוי)
- שדה כתובת יעד
- אזור העלאת תמונות (מומלץ)

**שלב 4: תזמון**
- 4 רמות דחיפות: מיד (+50%), בשעה הקרובה, במהלך היום, לתכנן ליום אחר
- שדות פרטי קשר

**שלב 5: סיכום**
- כל הפרטים (אופנוע, סיבה, מסלול, זמן)
- פירוט תשלום מלא
- תגיות אמון (ביטוח, GPS, תמונות לפני/אחרי)
- כפתור "אישור הזמנה"

**מסך אישור:**
- אייקון וי ירוק גדול
- "ההזמנה אושרה!"
- ETA banner סגול
- כפתור "התקשר לנהג"
- כפתור "צ'אט עם הנהג"
- מספר הזמנה

**מסך מעקב חי:**
- סטטוס בר עליון בסגול עם ETA חי + אנימציית פולס
- מפה חיה עם 2 פינים + מסלול נסיעה
- כרטיס נהג עם דירוג + כפתורי צ'אט/התקשרות
- ציר זמן עם 6 שלבי הנסיעה
- פירוט תשלום (מחיר נעול)
- כפתורי בטיחות: שתף מעקב + SOS
- כפתור ביטול חינם עד הגעה

## דרישות איכות קוד

1. **רכיבים פונקציונליים** עם React Hooks
2. **State management** ב-`useState` ו-`useContext` (אין צורך ב-Redux)
3. **PropTypes או TypeScript** לכל רכיב
4. **Accessibility:** `aria-labels`, `role`, ניווט מקלדת
5. **Responsive:** מובייל-first, עובד גם בדסקטופ
6. **Loading states** לכל פעולת רשת
7. **Empty states** לרשימות ריקות
8. **Error boundaries** סביב כל ממשק
