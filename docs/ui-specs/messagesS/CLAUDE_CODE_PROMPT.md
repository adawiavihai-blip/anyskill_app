# פרומפט ל-Claude Code — שדרוג מסך שיחה (Chat Screen) באפליקציית AnySkill

## הקשר כללי
אתה צריך לשדרג את מסך **השיחה הפרטית** (לא את רשימת ההודעות הכללית) באפליקציית AnySkill. זה המסך שנפתח **כשלוחצים על שיחה ספציפית** בין לקוח לנותן שירות מתוך לשונית "הודעות". האפליקציה בנויה ב-**Flutter** (main.dart.js) ותומכת ב-RTL (עברית).

**חשוב מאוד:** השינויים חלים **רק על מסך השיחה הספציפית** (הצ'אט בין שני משתמשים) — **לא** על רשימת השיחות הכללית בלשונית "הודעות".

לפני שתתחיל לקוד, עיין בקובץ המוקאפ המצורף (`messages_upgrade_final.html`) שמראה **איך התוצאה הסופית צריכה להיראות ולהתנהג**. הוא מכיל את כל האינטראקציות שצריך ליישם.

## 🎯 רשימת השינויים שצריך לבצע

### 1. ניקוי הכותרת (Header) של השיחה
**להסיר:**
- אייקון הטלפון 📞
- אייקון מצלמת הווידאו 🎥

**להשאיר:**
- כפתור חזור (←)
- תמונת פרופיל של המשתמש + נקודת סטטוס online
- שם המשתמש + סטטוס ("מחובר עכשיו" / "היה מחובר לפני X דקות")
- אייקון 3 נקודות (⋮) → פותח תפריט הגדרות שיחה

### 2. אווטאר (תמונת פרופיל) מסונכרן
- האווטאר בכותרת השיחה חייב להיות **מסונכרן עם תמונת הפרופיל האמיתית** של המשתמש (מה-backend/API).
- אם אין תמונה — fallback לאות הראשונה של השם עם רקע גרדיאנט סגול.
- להוסיף נקודה ירוקה (online indicator) בפינה, כאשר המשתמש מחובר.

### 3. הסרת כפתורי תשובות מהירות (Quick Replies)
**להסיר לחלוטין** את השורה שמעל תיבת הקלט שמכילה את הכפתורים:
- "סיימתי"
- "אני בדרך"
- "בקש תשלום"
- "שלח מיקום"
- "שלח תמונה"

### 4. הסרת מיקרופון / הודעות קוליות
**להסיר** את כפתור המיקרופון מתיבת הקלט — **לא תהיה יותר אפשרות לשלוח הודעות קוליות** בשיחה הזו.

### 5. תפריט סיכה 📎 (Attachment Menu)
הסיכה ליד שדה ההקלדה פותחת תפריט מוקפץ עם **4 אפשרויות בלבד**:

| אייקון | תווית | פעולה |
|--------|-------|-------|
| 📍 | **שלח מיקום** | פתיחת מסך מפה → שליחת מיקום נוכחי של המשתמש (GPS) |
| 📷 | **שלח תמונה** | פתיחת מצלמה עם אפשרות לצילום חדש **או** להעלאה מהגלריה |
| 🎥 | **שלח וידאו** | פתיחת מצלמה במצב וידאו **או** בחירת וידאו קיים מהגלריה |
| 💰 | **הצעת מחיר** | פתיחת מודאל הצעת מחיר (ראו סעיף 6) |

**דרישות טכניות:**
- למצלמה: להשתמש ב-device camera API של Flutter (`camera` package או `image_picker`).
- לגלריה: `image_picker` / `file_picker`.
- למיקום: `geolocator` + `google_maps_flutter` לתצוגה מקדימה של המפה.
- וידאו: לאפשר הקלטה עד 60 שניות ולשלוח עם thumbnail.
- חובה לבקש הרשאות (permissions) לפני שימוש במצלמה/מיקרופון/מיקום/גלריה — בכל פלטפורמה.

### 6. 💰 הצעת מחיר (Price Offer) — Airbnb-style

#### מה שנותן השירות רואה כשלוחץ על "הצעת מחיר":
מודאל מלמטה (Bottom Sheet) עם **3 שדות בלבד + בורר זמן**:

1. **שירות** — שדה טקסט (לדוגמה: "טיול עם הכלב 30 דק׳")
2. **מחיר רגיל של השירות (₪)** — שדה מספרי
3. **הנחה שתינתן ללקוח (₪)** — שדה מספרי
4. **זמן תוקף ההצעה** — 4 כפתורי בחירה (chips):
   - 30 דקות
   - שעה
   - 6 שעות
   - 24 שעות

**אסור להציג לנותן השירות:**
- ❌ את אחוז העמלה של האפליקציה (הוא כבר יודע אותה)
- ❌ מחשבון "כמה תקבל אחרי עמלה" (הוא מציע את המחיר — הוא יודע)

**כפתור שליחה:** "**שלח הצעה ללקוח**" (צבע סגול ראשי של האפליקציה).

#### מה שהלקוח רואה (הכרטיסייה שמגיעה אליו בצ'אט):
```
┌─────────────────────────────────┐
│  🏷️ הצעת מחיר מותאמת            │
│                                 │
│  💰 טיול עם הכלב · 30 דק׳         │
│  הצעה ממאמנת מאומתת             │
│  ─────────────────              │
│  מחיר רגיל         100.00 ₪ (מחוק) │
│  🎁 הנחה אישית    - 10.00 ₪     │
│  ─────────────────              │
│  💜 לתשלום         90.00 ₪      │
│                                 │
│  [ ✓ אישור ותשלום ] [ דחה ]     │
│                                 │
│  ⏱️ ההצעה תפוג בעוד 00:59:45    │
└─────────────────────────────────┘
```

**חשוב מאוד ללקוח:**
- ❌ **אסור** להציג את העמלה של AnySkill
- ❌ **אסור** להזכיר את עמלת האפליקציה
- ✅ להציג **רק**: מחיר רגיל (מחוק), הנחה, וסה״כ לתשלום

#### התנהגות ההצעה:
- **טיימר תפוגה חי** (countdown) שמתעדכן כל שנייה
- **צבעי טיימר דינמיים:**
  - סגול (רגיל) — יותר מ-5 דקות נשארו
  - 🟡 כתום — בין 1-5 דקות נשארו
  - 🔴 אדום — פחות מדקה
- **כשהזמן נגמר:**
  - הכרטיסייה הופכת לאפורה (opacity 65%)
  - הכפתורים נעלמים
  - מוצג באדג' "⏰ ההצעה פגה"
  - לחיצה על "אישור ותשלום" אחרי שפג — לא עושה כלום
- **כשלוחצים "אישור ותשלום":**
  - מחיקת הכפתורים והטיימר
  - הצגת באדג' ירוק "✅ ההצעה אושרה והתשלום בוצע בהצלחה"
  - **חיוב הלקוח** בסכום המוצג (90 ₪ בדוגמה)
  - העברה ל-**חשבון נאמנות (escrow)** של AnySkill — הכסף לא עובר למאמן עד סיום השירות
  - **עמלת AnySkill (15%) מנוכה מהסכום שהמאמן יקבל** — לא מהסכום שהלקוח משלם
  - חישוב לדוגמה:
    - לקוח משלם: 90 ₪
    - עמלת AnySkill: 15% × 90 = 13.50 ₪
    - מאמן מקבל: 90 - 13.50 = **76.50 ₪** (אחרי סיום השירות)

#### חשוב — סנכרון ואבטחה:
- **שיעור העמלה (15%) מגיע מה-backend ונעול על ידי המערכת** — נותן השירות לא יכול לערוך או לראות אותו ב-UI של ההצעה.
- **ההצעה חייבת להיות מסונכרנת:** אישור/דחייה/תפוגה חייבים להתעדכן אצל שני הצדדים בזמן אמת (WebSocket / Firebase / Pusher).
- הצעה לא ניתנת לאישור פעמיים.
- לאחר תפוגה — גם ב-backend ההצעה מסומנת כ-`expired` ולא ניתנת לאישור.

### 7. 🌍 תרגום אוטומטי דו-צדדי (שני הצדדים)

#### פעולה:
- כשמגיעה הודעה בשפה שונה מהשפה המוגדרת של המשתמש — **המערכת מזהה את השפה אוטומטית** (Google Cloud Translation API או DeepL).
- התרגום **מופיע אוטומטית מתחת להודעה המקורית** אם המשתמש הפעיל "תרגום אוטומטי" בהגדרות.
- הטקסט המקורי נשאר גלוי (מעומעם בשקיפות ~60%), התרגום מופיע בקופסה נפרדת מתחתיו עם הכותרת "🌐 תורגם אוטומטית מ[שפת המקור]".

#### UI של הודעה מתורגמת:
```
┌─────────────────────────────────┐
│ ሰላም! የውሻዎን የእግር ጉዞ ጥያቄ አይቻለሁ   │ ← מקור (מעומעם)
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─        │
│ 🌐 תורגם אוטומטית מאמהרית       │
│ שלום! ראיתי את הבקשה שלך לטיול  │ ← תרגום
│ עם הכלב 🐕                     │
│                                 │
│ [🌐 הסתר תרגום] [AM]           │ ← כפתור + באדג' שפה
└─────────────────────────────────┘
```

#### הגדרות תרגום (בתוך תפריט ⚙️ של השיחה):

| הגדרה | אפשרויות |
|--------|----------|
| **תרגום אוטומטי** | Toggle (מופעל/כבוי) |
| **השפה שלי** | Dropdown עם לפחות: עברית, אנגלית, ערבית, רוסית, אמהרית, ספרדית, צרפתית |

#### דרישות טכניות:
- **זיהוי שפה אוטומטי** על כל הודעה נכנסת.
- **אם שפת ההודעה = השפה של המשתמש** → לא לתרגם ולא להציג באדג'.
- **שני הצדדים יכולים להפעיל/לכבות** את התרגום באופן עצמאי (כל אחד בהגדרות שלו). כשהלקוח כיבה אצלו — זה **לא** משפיע על מה שנותן השירות רואה.
- כל משתמש יכול לבחור שפת יעד משלו.
- **תרגום ידני של הודעה ספציפית:** גם כשהתרגום האוטומטי כבוי, כל הודעה זרה תציג כפתור קטן "🌐 הצג תרגום" — לחיצה עליו מתרגמת אותה בלבד.
- תרגומים נשמרים ב-cache מקומי כדי לא לתרגם שוב ושוב את אותה הודעה.

### 8. 🌙 Dark Mode

#### 3 מצבים בהגדרות:
1. ☀️ **בהיר** — תמיד בהיר
2. 🌙 **כהה** — תמיד כהה
3. 🕐 **אוטומטי** (ברירת מחדל) — **כהה בין 19:00-07:00, בהיר בשאר היום**

#### דרישות:
- **כל המסך** צריך להיות מותאם לשני המצבים: רקעים, טקסטים, גבולות, כפתורים, כרטיסיות, מודאלים.
- **לא** להוסיף כפתור צף חיצוני או אייקון ירח/שמש — הגישה למצב הכהה היא **רק** דרך תפריט ההגדרות (3 הנקודות בכותרת).
- **המצב האוטומטי** חייב לבדוק את השעה **כל דקה** ולעדכן בזמן אמת אם עובר 19:00 או 07:00 בזמן שהשיחה פתוחה.
- להשתמש ב-Flutter `ThemeData` עם `ThemeMode.system`, `ThemeMode.light`, `ThemeMode.dark` — או theme provider מותאם.
- פלטה:
  - **סגול ראשי**: `#4F46E5` (בהיר) / `#818CF8` (כהה)
  - **רקע**: `#F8FAFC` (בהיר) / `#0F172A` (כהה)
  - **משטחי כרטיסיות**: `#FFFFFF` (בהיר) / `#1E293B` (כהה)

### 9. תפריט הגדרות שיחה (⚙️ דרך 3 הנקודות בכותרת)
מודאל מלמטה שמכיל:

**חלק א' — תרגום:**
- Toggle "תרגום אוטומטי" (on/off)

**חלק ב' — מראה:**
- 3 כפתורי בחירה: בהיר / כהה / אוטומטי

**חלק ג' — כפתור שמירה:**
- "שמור וסגור" (סגול, ראשי)

6. ❌ הצגת עמלת AnySkill באף מקום ב-UI של ההצעה (לא בצד נותן השירות ולא בצד הלקוח)
7. ❌ מחשבון "כמה תקבל אחרי עמלה" בצד נותן השירות

- כרטיסיית הצעה עם טיימר חי
- תרגום הודעות מאמהרית לעברית עם כפתור toggle

**השתמש בו כ-source of truth לעיצוב ולהתנהגות.**

## 🎨 עקרונות עיצוב

- **פונט:** Heebo (Google Fonts) — עברית ברורה, תומך RTL מצוין
- **צבעים ראשיים:**
  - `--primary: #4F46E5` (סגול אינדיגו)
  - `--primary-light: #818CF8`
  - `--primary-dark: #3730A3`
  - `--accent: #10B981` (ירוק — לסטטוסים חיוביים)
  - `--danger: #EF4444` (אדום — לאזהרות/תפוגה)
- **פינות מעוגלות:** 12-20px ברוב המקומות
- **צללים עדינים:** לא כבדים
- **אנימציות:** מעבר חלק בין דארק/לייט (~400ms), fade-in להודעות חדשות, pulse לנקודת online

## 📌 סדר ביצוע מומלץ

1. **תחילה**: הסרות — אייקוני טלפון, מיקרופון, כפתורי quick replies
2. **שנית**: עיצוב חדש של הכותרת (אווטאר מסונכרן + סטטוס)
3. **שלישית**: תפריט סיכה עם 4 אפשרויות (מקום, תמונה, וידאו, הצעה)
4. **רביעית**: מסכי מצלמה/גלריה/מיקום + התחברות להרשאות ו-APIs
5. **חמישית**: הצעת מחיר — מודאל שליחה + כרטיסיית הצעה + backend logic + טיימר
6. **שישית**: תרגום אוטומטי — זיהוי שפה + תרגום + UI + cache + הגדרות
7. **שביעית**: Dark Mode — theme provider + החלת סגנונות לכל הרכיבים + בורר בהגדרות
8. **שמינית**: QA — מעבר על כל ה-Checklist
