## 6. Deploy commands (run in this order)

```powershell
# 1. Composite indexes (1-3 min build time in console)
firebase deploy --only firestore:indexes

# 2. Security rules (instant)
firebase deploy --only firestore:rules

# 3. Cloud Functions (2-4 min)
firebase deploy --only "functions:updateCategoryAnalytics,functions:refreshCategoryAnalyticsNow,functions:logAdminAction,functions:undoAdminAction,functions:backfillCategoriesV3"

# 4. Web hosting (Phase B onwards)
flutter build web --release
firebase deploy --only hosting
```

### One-time post-deploy step

After CFs are deployed, invoke the backfill ONCE from the Firebase Console:
1. Console → Functions → `backfillCategoriesV3` → "Force run" → payload `{}`
2. Expected: `{ok: true, scanned: ~77, initialized: ~77, skipped: 0}`
3. Verify in Firestore Console: open any category doc, confirm `admin_meta` field exists with `is_pinned: false, is_hidden: false, last_edited_action: 'backfill'`

The scheduled `updateCategoryAnalytics` runs every 15 min on its own — no need to invoke manually.

## 8. Known limitations (deferred to Phase F+)

These are intentional Phase D/E omissions, not bugs:

- **Tab "תתי" in Edit dialog is read-only.** Sub-category editing happens from the main grid. A future PR can wire inline create/edit there.
- **"Move to parent" bulk action not built.** Was in spec but lower priority; the EditCategoryDialog "details" tab will gain a parent-dropdown in a future PR.
- **`promoted_banners` is read-only mirror.** Editing a banner doc here does NOT yet flow to `home_tab.dart` rendering. Migration is a separate PR per Q5-A decision.
- **`views_30d` / `clicks_30d` show "—" placeholders.** No tracking infra exists yet. Per Q4-B+C, build only when DAU justifies the write cost.
- **Reorder undo throws a friendly error.** Snapshotting a 50-doc batch into `payload_before` is too heavy. Manual re-sort is the workaround.
- **Dark mode not supported.** Light-only matches Vault §29 + Monetization §31 admin convention.

```powershell
# Option 1: Revert the feature flag (instant — no deploy needed)
# Edit lib/screens/categories_v3/feature_flag.dart, set _whitelistedUids = {}
# Hot-reload — admin sees legacy tab immediately
```

```powershell
# Option 2: Revert all 5 commits (full rollback)
git revert ffe68d1 4918d5f fdb7ec3 3f16e52 <phase-e-sha>
```

```powershell
# Option 3: Disable Cloud Functions (stops analytics aggregation)
gcloud functions delete updateCategoryAnalytics --region us-central1
# Existing categories.analytics fields stay intact (frozen at last update)
```

The feature flag is the cheapest rollback — all infrastructure (CFs, indexes, rules, schema fields) is additive and harmless if the v3 UI never renders.
